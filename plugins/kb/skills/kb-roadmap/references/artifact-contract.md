# Reference: artifact output contract

Every `/kb roadmap` run emits three files with the same base name under the scope-specific directory:

```
<output-dir>/<scope>/roadmap-<YYYY-MM-DD>.md
<output-dir>/<scope>/roadmap-<YYYY-MM-DD>.html
<output-dir>/<scope>/roadmap-<YYYY-MM-DD>.json
```

For roll-up scopes, the base name includes the scope explicitly: `roadmap-<scope>-<YYYY-MM-DD>.{md,html,json}`. See `references/folder-layout.md` for the full folder layout and the detail-vs-roll-up section split.

## JSON sidecar (authoritative)

The JSON is the source of content; MD and HTML are renderings. Schema: `templates/roadmap.json.schema.json`.

Top-level shape:

```json
{
  "version": "0.2.0",
  "generated_at": "2026-04-21T14:30:00Z",
  "scope": "product-a",
  "timeframe": { "kind": "since", "since": "2026-04-01" },
  "plan_sources": [ { "name": "...", "adapter": "...", "items": [...] } ],
  "delivery_sources": [ { "name": "...", "adapter": "...", "items": [...] } ],
  "correlations": [ { "plan_id": "...", "delivery_id": "...", "tier": 2, ... } ],
  "mismatches": {
    "delivered-unplanned": [...],
    "planned-undelivered": [...],
    "traceability-gap": [...],
    "stalled-in-progress": [...]
  },
  "sections": {
    "A": {...}, "B": {...}, "C": {...},
    "D": {...}, "E": {...}, "F": {...}, "G": {...}
  },
  "counts": {
    "plan_items": 42,
    "delivery_items": 118,
    "matches_by_tier": { "1": 12, "2": 8, "3": 5, "4": 3 },
    "mismatches_total": 17
  },
  "warnings": []
}
```

## Markdown report

Human-readable. Seven sections in order, each with a heading and a table or list. Always emits the section heading even if empty.

```markdown
# Roadmap — <scope> — <date>

> <timeframe description>
> <counts summary line>

## A. Plan baseline
...

## B. Delivery baseline
...

## C. Correlation matrix
<table: tier | count | examples>

## D. Delta since <since-date>
...

## E. Mismatch findings
### Delivered, unplanned
### Planned, undelivered
### Traceability gaps
### Stalled

## F. Forward plan
...

## G. Decisions needed
...

---
Generated by kb-roadmap v<version> · <timestamp>
```

## HTML report

Renders from the JSON into the template at `templates/roadmap.html.hbs`. Contract:

- Seven section anchors `#section-a` through `#section-g`
- Sticky nav listing the seven sections
- Theme toggle (light / dark / auto), persisted in localStorage
- Optional logo slot in header — left-side, SVG, respects theme via `prefers-color-scheme` + `[data-theme]`
- Optional watermark below the title (`{version} · {date}`)
- WCAG AA contrast in both themes
- Print-friendly fallback (`@media print` hides nav and theme toggle, collapses to single column)
- No external network dependencies — fonts, icons, styles all inline
- No JavaScript required for content visibility (JS is progressive enhancement for theme toggle and nav scroll)

## Styling tokens

The template reads CSS custom properties from the adopter's `.kb-config/artifacts.yaml` `html-template.tokens` setting:

- `neutral` (default) — muted greys, high-contrast accent
- A file path — a CSS file that overrides the token block (adopters supply brand palettes here)

The skill itself ships no brand colors or logos. Adopters that want a branded artifact provide:

```yaml
html-template:
  base: kb-roadmap/templates/roadmap.html
  tokens: .kb-references/brand/tokens.css
  logo:
    light: .kb-references/brand/logo-light.svg
    dark: .kb-references/brand/logo-dark.svg
```

## Versioning

The `version` field in the JSON matches the skill's SKILL.md `version`. Schema changes bump this. Older JSON artifacts remain readable across minor versions within a major.
